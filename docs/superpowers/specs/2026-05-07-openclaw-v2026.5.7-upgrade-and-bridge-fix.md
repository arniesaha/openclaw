## Track 2 — Bridge-coverage OpenClaw patch

### Problem

OpenClaw's isolated-cron and heartbeat agent paths run LLM calls without emitting the `message.queued` diagnostic event. The agentweave-bridge plugin attaches root spans to that event, so isolated-session calls reach the proxy but get bucketed under the static `X-AgentWeave-Session-Id: nix-main` header. Result: 1,732 span rows but only 4 distinct `session.id` values for 63 native-shipper sessions.

`logMessageQueued` is currently called only from `src/auto-reply/reply/dispatch-from-config.ts:417` (channel dispatch) and `src/agents/pi-embedded-runner/runs.ts:75,89` (embedded runner).

### Approach

- Cut `fix/cron-isolated-message-queued` from `agentweave/v2026.5.7` post-upgrade.
- Add `logMessageQueued({ sessionKey, channel: "cron", source: "cron-isolated" })` at the agent-turn entry in `src/cron/isolated-agent/run.ts`. Match the existing call shape (sessionId + sessionKey + source).
- Mirror for the heartbeat-runner isolated path in `src/infra/heartbeat-runner.ts` near line 1266 if it shares the same isolated-session model. Confirm during implementation.
- Add a paired `logMessageProcessed` on session completion so the bridge ends the root span cleanly.

### Validation on Nix

- Build, restart gateway.
- Trigger one isolated cron job manually: `openclaw cron run <portfolio-briefing-id>` (registered at `src/cli/cron-cli/register.cron-simple.ts:160`; `--due` to gate, default forces).
- Tail journal for `[agentweave-bridge] event: message.queued source: cron-isolated`.
- Query Tempo for a span carrying the new session UUID; confirm `prov.session.id` is the run UUID, not `nix-main`.
- Compare distinct session count over a short window before/after the patch on the same daily cron schedule.

### Upstream PR

- Open against `openclaw/openclaw` once validated.
- Include a test in `src/cron/` (or alongside `src/agents/pi-embedded-runner/runs.test.*` if that's the established pattern) asserting the event fires for isolated-agent execution.
- Reference issues `arniesaha/agentweave#180`, `#186`, and the OpenClaw-side complement (open a paired tracking issue if missing).
